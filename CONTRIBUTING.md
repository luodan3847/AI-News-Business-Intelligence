# Contributing

Thanks for taking an interest in Claude Code Guide.

This repository is currently maintained as a focused personal project, so the best contributions are small, clear improvements that strengthen the product, documentation, or workflow quality.

## Good contributions

- fix bugs or polish the public-facing experience
- improve documentation clarity
- tighten accessibility, structure, or naming
- refine automation scripts without making the project harder to maintain

## Before opening a change

- read the README for project context
- keep the project professional and easy to understand
- preserve the separation between UI, data, scripts, and documentation
- prefer clear, incremental changes over broad rewrites

## Development checks

Run the main verification commands before submitting changes:

```bash
npm run typecheck
npm run build
```

Or run the combined check:

```bash
npm run check
```

## Style expectations

- keep copy concise and public-facing
- favor structure and clarity over feature sprawl
- document decisions when they improve understanding for future readers

## Pull requests

When opening a pull request, include:

- what changed
- why it improves the project
- any screenshots if the UI changed
